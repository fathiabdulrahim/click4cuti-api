Tab Navigation — Design Options

The admin user-detail page has 15 tabs (Personal, Job, Approval, Contact, Dependents, Work Experience, Career, Payroll, Files, Training, Assets, Claim Policy, Claim Balances, Claim History, Conduct). On a typical laptop screen ~9 fit; the rest are hidden behind horizontal scroll with no visual cue. Below are 5 patterns ranked by fit for this product.

Patterns considered

  1. Sidebar nav with grouped sections Recommended
  2. Top tabs grouped into sections (chips)
  3. Visible tabs + "More" overflow dropdown
  4. Scroll affordances (arrows + edge fade)
  5. Search-driven chips (Cmd+K)

Recommendation: Option 1 — Sidebar Nav

With 15 sections, top tabs hit their ceiling. BambooHR, Rippling, and Personio all use a left vertical nav for employee detail pages — it scales infinitely, groups related sections, and matches the mental model of "drill into one employee's record." For Click4Cuti it also gives breathing room to the data-dense forms (PersonalDetail has 14 fields, Payroll has 3 sub-cards).

If you want a quicker, lower-risk fix, Option 3 (More dropdown) keeps the current top-tab pattern with overflow handled — minimal refactor.

Keep tabs at the top, but break the long strip into labelled groups using pill chips. Reduces visual length by giving the eye semantic anchors.

  • Familiar pattern from Notion, Linear, GitHub

  • Two clicks to reach lesser-used tabs
Effort: Low-Medium (~3 hours) Mobile: All tabs in dropdown
4

Scroll Affordances (Arrows + Edge Fade)

Keep the current scrolling tabs, but add a fade-out gradient on the right edge plus clickable chevron arrows that scroll by ~200px. The gradient appears only when there are tabs hidden.

Pros

  • Smallest code change — just add scroll arrows + gradient overlay
  • Gradient is a strong "there's more here" affordance
  • Auto-scrolls active tab into view on selection

Cons

  • Still requires the user to scroll to find a tab — not direct
  • No grouping or hierarchy — all 15 tabs feel equal
  • Doesn't fix the underlying "too many tabs" problem
Effort: Low (~1 hour) Mobile: Same as desktop
5

Search-driven Chips (Cmd+K)

Show frequently-used tabs as chips, add a global Cmd+K search to jump to any tab. Power-user pattern from Linear, Vercel.

Pros

  • All tabs visible at once, no scroll
  • Cmd+K is a power-user multiplier
  • "Hot" chips can highlight tabs with attention items (e.g. expired training)
  • Adapts to mobile cleanly (chips wrap)

Cons

  • Chip cloud takes more vertical space than a tab bar
  • Less hierarchical — all chips feel equal
  • Cmd+K requires keyboard-shortcut implementation (cmdk or similar)
  • HR admins may not be power-users; ⌘K mostly discoverable only with hint
Effort: Medium (~5 hours; needs cmdk lib) Mobile: Chips wrap; bottom-sheet search

Quickest path: Option 4 (scroll arrows + gradient) — solves the "not obvious" problem in ~1 hour without changing the IA. Best long-term: Option 1 (sidebar) — matches industry leaders and scales as Click4Cuti adds more sections. Hybrid worth considering: Option 3 with grouped dropdown (uses tab strip on wide screens, sidebar/dropdown on narrow).
